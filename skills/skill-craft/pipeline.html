Phase 1: DIAGNOSING

1
Audit Pass — 13 Lenses skill-audit ▼
Run /skill-audit on target skill — produces structured findings across 13 lenses
Reference Integrity, Process Enforcement, Command Discipline, Structural Justification
Template System, Model Variance Risk, Contract Completeness, Skill Contract Consistency
Mechanism Leakage, Question Strategy, Operational Resilience, Assurance Strategy, Non-Goals Clarity
target_skill_path → /skill-audit → audit_findings (Finding[])
2
Craft Lens — Validation Checklist skill-development ▼
Run craft_lens_enforcer — codified skill-development validation checklist
Imperative/infinitive form check (no second person "you should")
Third-person trigger description check (This skill should be used when...)
SKILL.md body line count (warn >300, block >500 lines)
Progressive disclosure verification (references/ exist and are referenced)
Resource directory structure validation
target_skill_path → craft_lens_enforcer → craft_issues (Finding[])
↓

3

Phase 3: EXECUTING

4
Invoke Subskills by Priority ▼
🛠 /skill-ship
Implementation fixes: hooks, validators, integrations, operational resilience contracts
🚀 skill-creator
Eval iteration: run_eval → grader → benchmark → analyst → feedback → repeat
🌱 skill-development
Craft structure: progressive disclosure, imperative form, trigger phrasing
🔍 /skill-audit
Strategic re-diagnosis when scope or enforcement model has drifted
action_list → invoke_by_priority(ship | creator | development | audit) → phase_results
↓

Phase 4: EVALUATING

5
Fidelity Measurement fidelity_tracker ▼
Trigger accuracy: % of expected triggers that fired in Bruce's environment
Outcome accuracy: % of outputs matching expected behavioral patterns
Degradation delta: fidelity change vs last baseline (detect drift over time)
Environment notes: context/state issues that caused failures
target_skill_path + eval_set → fidelity_tracker.run() → fidelity_score → pass|fail
Loop-back: If fidelity_score fails threshold, loop back to Phase 1 with updated state. This is the "won't be used properly" gate — not just structural correctness, actual working fidelity.
↓

Phase 5: GATING

6
Certification Gate CertificationGate ▼
Context size
validate_context_size()SKILL.md body <500 lines, references loaded as-needed
↘
Frontmatter
name + description presentRequired fields per skill-frontmatter-fields.md
↘
Trigger accuracy
description triggers match usageNo hallucinated flags, no drift from body
Fail route: If cert gate fails, route to appropriate skill for repair — do NOT exit loop. Only exit when both fidelity_score passes AND cert gate passes.
target_skill_path → CertificationGate.check() → pass|fail → route|exit
↓

Sub-loops via /tilldone

7
Delegated Sub-loops ▼
🔄 Eval iteration
"Run eval until all queries pass or max 5 iterations" — delegated to /tilldone
🔍 Audit pass
"Run audit until no HIGH findings remain or max 3 passes" — delegated to /tilldone
skill-craft invokes /tilldone for sub-loops rather than duplicating loop-control logic. This keeps skill-craft as an orchestrator, not a loop controller.
