Workflow Steps

Step 1 — Read SKILL.md

▼

Read the target skill's SKILL.md completely. Extract frontmatter, workflow_steps, description, triggers, routing, terminal states, and artifacts.

Step 2 — Build Artifact Plan

▼

Design the page structure from the workflow model. Emit artifact-plan.json with section order, content bindings, and ui_config.

Step 3 — Design Mermaid

▼

Generate the Mermaid diagram from the normalized workflow model — not from prose. Apply layout rules, color-coded classDefs, and edge labels.

Step 4 — Mermaid Critic

▼

Validate the diagram: zero edge crossings, all nodes labeled, all edges labeled, readable at 50/100/150% zoom. Gate: crossings==0 AND syntax valid.

Step 5 — Emit HTML

▼

Assemble index.html from template skeleton files. Fill content slots from artifact-plan.json. CSS/JS come from templates only — never generated from prose.

Step 6 — Static Validate

▼

Run S1–S19 regex checks on index.html: tocToggle sibling, nav#toc class, CSS properties, element presence, content-model binding.

Step 7 — Runtime Validate

▼

Launch browser via browser-harness. Verify page loads, TOC visible by default, step header present, search input exists, Mermaid pre present, resize handle present, zoom controls, copy buttons.

Step 8 — External Critic

▼

Run an independent LLM critic against the HTML. Contract checks: no_invented_routes, toc_initial_state_synced, toc_handler_atomic, html_represents_workflow_model, steps_complete, no_internal_policy_prose.

Step 9 — Emit Proof Metadata

▼

Write workflow-model.json, artifact-proof.json, and diagram.mmd to the artifacts directory. Skill terminates successfully.
