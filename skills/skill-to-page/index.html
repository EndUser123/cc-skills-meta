Workflow Steps

Step 1 — Read Skill Source

▼

Read the target skill's SKILL.md. Extract frontmatter, workflow_steps, description, triggers, prose routing, checklists/gating questions, terminal states, artifacts, sub-skills, and verification expectations.

Inputs
P:/.claude/skills/{target}/SKILL.md
Outputs
raw_skill_content

Step 2 — Extract Workflow Model

▼

Build a normalized internal workflow model. The model is the source of truth for Mermaid generation, accordion sections, TOC, verification checks, and proof metadata.

Inputs
raw_skill_content
Outputs
workflow-model.json

Step 3 — Detect Source Gaps

▼

Cross-check for prose-only routing, checklist branching, conditional steps shown as unconditional, missing descriptions, unrepresented terminal states or artifacts, and naming mismatches.

Gateambiguities.length === 0 OR ambiguities_recorded === true
Inputs
raw_skill_content, workflow-model.json
Outputs
workflow-model.json (updated)

Step 4 — Design Mermaid Diagram

▼

Generate Mermaid from the workflow model. Rules: flowchart TD, curve:'basis', nodeSpacing:40, rankSpacing:50, ≤40 char labels, classDefs by node type.

Inputs
workflow-model.json
Outputs
diagram.mmd

Step 5 — Mermaid Critic Review

Gate ▼

Spawn agent:general-purpose to validate diagram. Checks: traceability, crossings (must be 0), label clarity, edge labeling, readability at 50% zoom, syntax validity, coverage of all model elements, theme-safe colors.

Gatecrossings==0 AND syntax_errors==[] AND legibility_score>=0.8 AND missing_steps==[] AND missing_route_outs==[] AND missing_terminal_states==[] AND dark_theme_contrast_ok==true AND light_theme_text_readable==true
Inputs
diagram.mmd, workflow-model.json

Step 6 — Generate HTML

▼

Build index.html from the workflow model: hero card, quick facts, Mermaid diagram, accordion per step, route-outs section, terminal states section, artifact cards with copy-to-clipboard paths, proof summary, theme toggle, search UI, responsive layout.

Inputs
workflow-model.json, diagram.mmd
Outputs
index.html

Step 7 — Browser Verify Artifact

Orchestrator Only ▼

Orchestrator-only CDP verification. 13 checks: file exists, Mermaid renders, TOC links resolve, TOC initial state synced, content reflow, theme toggle preserves viewport, zoom/reset, drag-to-pan, wheel zoom, search finds sections, accordion works, no duplicate listeners, no console errors.

Timeout
120000ms total, 30s per check
Inputs
index.html

Step 8 — Artifact Critic Review

Gate ▼

Spawn agent:general-purpose to verify fidelity. Checks: HTML represents workflow model, step coverage, no invented routes (blocking), TOC initial state sync (blocking), TOC handler atomicity (blocking).

Gateno_invented_routes==true AND toc_initial_state_synced==true AND toc_handler_atomic==true
Inputs
index.html, workflow-model.json, diagram.mmd

Step 9 — Emit Proof Metadata

▼

Write workflow-model.json, artifact-proof.json, and diagram.mmd to the artifacts directory.

Inputs
workflow-model.json, verification_result, critic_results
Outputs
artifact-proof.json, workflow-model.json, diagram.mmd
